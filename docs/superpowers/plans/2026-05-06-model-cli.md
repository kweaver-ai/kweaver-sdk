# Model Management CLI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 `kweaver-sdk` 中提供 `kweaver model` 命令组，承载 mf-model-manager 的大模型 (LLM) 与小模型 (embedding / reranker) 元数据 CRUD、连通性测试，并支持小模型的 **adapter 适配代码** 注册流程，简化模型部署对接。

**Out of scope（本期不做）：**
- LLM 调用代理（`/chat/completions` 转发）
- model quota / prompt / model audit 管理
- Python SDK parity（仅 TS CLI；Python 留 follow-up）
- 在 SDK 端执行 `adapter_code`；SDK 仅做客户端侧字段互斥校验后透传

**Architecture:**

```
┌──────────────────────────────────────────────────────┐
│  CLI (commands/model.ts)                             │
│    kweaver model llm   list|get|add|edit|delete|test │
│    kweaver model small list|get|add|edit|delete|test │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│  Resource (resources/models.ts)                      │
│    client.models.llm.* / client.models.small.*       │
└────────────────────────┬─────────────────────────────┘
                         │
┌────────────────────────▼─────────────────────────────┐
│  HTTP (api/models.ts)                                │
│    fetch + buildHeaders, /api/mf-model-manager/v1/*  │
└──────────────────────────────────────────────────────┘
```

**Tech Stack:** TypeScript, Node.js (>=22), `fetch`, 现有 `buildHeaders` / `with401RefreshRetry` / `formatHttpError`

## CLI 形状

```text
kweaver model llm   list   [--keyword X] [--type T] [--series S] [--page N] [--limit N=30] [--json] [-bd value]
kweaver model llm   get    <model_id> [--json] [-bd value]
kweaver model llm   add    --body-file <path.json>          [-bd value]
kweaver model llm   edit   --body-file <path.json>          [-bd value]
kweaver model llm   delete <model_id> [<model_id> ...] [-y] [-bd value]
kweaver model llm   test   --body-file <path.json>          [-bd value]

kweaver model small list   [--keyword X] [--type embedding|reranker] [--series S]
                           [--page N] [--limit N=30] [--json] [-bd value]
kweaver model small get    <model_id> [--json] [-bd value]
kweaver model small add    --name N --type embedding|reranker
                           ( --model-config-file path.json | --adapter --adapter-code-file path.py )
                           [--batch-size N] [--max-tokens N] [--embedding-dim N] [--json] [-bd value]
kweaver model small edit   <model_id> [...same flags as add, all optional]
kweaver model small delete <model_id> [-y] [-bd value]
kweaver model small test   ( <model_id> | --name N --type T ( --model-config-file ... | --adapter --adapter-code-file ... ) )
```

**Defaults & 约定**

- 列表类 `--limit` **默认 30**（仓库 AGENTS.md 强约定）；映射到后端 `size`，`--page` 默认 1。
- `--json` / `--pretty` 与全局命令一致。
- `delete` 命令带 `-y` 跳过确认；批量删除以传多个 id 实现。
- `--adapter-code-file` 读取本地 `.py` 文件内容塞入请求体 `adapter_code`。
- 客户端先做 **「`model_config` 与 `adapter` 互斥」** 校验，失败立刻报错（不打到后端再读 422）。
- 错误统一走 `formatHttpError`；写操作走 `with401RefreshRetry`。

## 风险 & 决策记录

1. **base URL**：是否所有部署都把 `/api/mf-model-manager` 挂到平台同一域？  
   - 默认假设 yes；不一致时通过 `--mf-base-url` / `KWEAVER_MF_MODEL_MANAGER_URL` 兜底。开工前与运维确认一次。

2. **`AddExternalSmallModel` 的 `model_config` 校验与 adapter 互斥**：后端 `model_config` 校验器检查必有 `api_url/api_model`，但 adapter 路径下不应填 `model_config`。已知后端 root_validator 已允许 `model_config={}` + adapter 组合（互斥校验先于子字段校验时通过空 dict）。**实现里 SDK 默认传 `model_config: {}`**；如发现新版本后端拒绝，再切到不传 `model_config` 字段。

3. **批量 delete**：后端 small-model `delete` 当前接收 `dict`，多 id 行为待 DAO 层确认。MVP 先按**单 id 逐个调用**实现批量；优化项放下个迭代。

4. **adapter_code 安全**：SDK 不执行任何 `adapter_code`，只透传字符串。reference 文档需明显标注「adapter 在服务端 `exec`，禁止上传不可信代码」。

5. **Python SDK parity**：本期不做。下一期单独 PR 镜像 `api/models.ts`。
